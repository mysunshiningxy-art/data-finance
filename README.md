# 全渠道基金管理系统

一站式管理多渠道基金持仓，支持截图识别导入、净值自动同步、收益追踪、行业分析与 AI 智能解读。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 截图 OCR 导入 | 上传支付宝/京东/微信基金持仓截图，自动识别基金名称、金额、收益 |
| 手动录入 | 表单添加交易记录（买入/卖出/分红） |
| 多渠道管理 | 支持支付宝、京东金融、微信理财通，统一汇总 |
| 净值自动同步 | 从公开数据源获取最新净值，实时计算市值和收益 |
| 持仓总览 | 按渠道/行业筛选、排序、下载 CSV |
| 持仓分析 | 7 种可视化图表：渠道分布、行业分布、持仓占比、盈亏排行等 |
| 行业分析 | 自动识别基金所属行业，支持行业维度的筛选和统计 |
| AI 智能解读 | 一键分析投资风格、配置合理性，给出调仓建议和综合评分 |
| 交易记录 | 完整交易历史查询 |
| CSV 导出 | 持仓明细一键下载 |

## 安装与启动

### 环境要求

- Python 3.11+
- macOS / Linux / Windows

### 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/mysunshiningxy-art/data-finance.git
cd data-finance

# 2. 安装依赖
pip install -r requirements.txt

# 3. 创建配置文件
cp config.yaml.example config.yaml
```

### 启动

```bash
python -m uvicorn app:app --host 0.0.0.0 --port 8000
```

打开浏览器访问 **http://localhost:8000**

## 使用指南

### 一、截图导入（推荐）

1. 打开手机上的支付宝/京东/微信基金持仓页面，截图保存
2. 在系统中点击「截图导入」Tab
3. 选择对应渠道（支付宝 / 京东金融 / 微信理财通）
4. 上传截图（支持多选图片、选择文件夹、拖拽上传）
5. 点击「开始解析」，系统自动识别基金名称、金额、收益
6. 核对解析结果，如有误差可手动修改
7. 选择截图日期，点击「确认导入」

> 截图要求：使用手机原图，不要裁剪或压缩，确保基金列表完整显示。

> 份额计算说明：系统使用截图日期前一个交易日的净值计算持有份额。

### 二、手动录入

1. 点击「手动录入」Tab
2. 填写基金代码、渠道、交易日期、交易金额
3. 系统自动查询基金名称和净值
4. 点击提交

### 三、查看持仓

在「总览」页面可以：

- **查看汇总** — 总成本、总市值、总盈亏、总收益率
- **渠道统计** — 点击渠道卡片可快速筛选
- **行业筛选** — 按行业过滤持仓
- **排序** — 点击表头按份额/成本/净值/市值/盈亏/收益率排序
- **同步净值** — 点击「同步全部净值」更新所有基金最新净值
- **下载** — 点击「下载」导出 CSV 文件
- **删除** — 删除单只基金持仓或一键清空

### 四、持仓分析

在「持仓分析」页面可以：

- **关键指标** — 最大持仓、最高/最低收益率、最大盈利/亏损
- **分布图表** — 渠道分布、行业分布、持仓占比（环形图）
- **收益排行** — 盈亏金额排行、收益率排行（柱状图）
- **成本对比** — 成本 vs 市值对比图
- **同步行业** — 点击「同步行业分类」自动识别所有基金行业

### 五、AI 智能解读

在持仓分析页面底部：

1. 点击「开始分析」按钮
2. 等待 10-20 秒，AI 将从四个维度分析：
   - **持仓风格画像** — 判断你的投资风格（激进/稳健/保守/均衡）
   - **资产配置分析** — 股债配比、行业集中度是否合理
   - **持仓诊断** — 哪些基金表现好/差，存在哪些风险
   - **调仓建议** — 具体的增减配方向和理由
3. 最后给出 1-10 分综合评分

> AI 功能需要在 config.yaml 中配置 API Key，详见下方「配置说明」。

## 配置说明

编辑项目根目录下的 `config.yaml`：

```yaml
database:
  url: "sqlite:///fund_manager.db"

channels:
  alipay:
    name: "支付宝"
    enabled: true

# AI 分析功能（可选）
ai:
  api_key: "sk-你的API密钥"
  base_url: "https://api.deepseek.com"
  model: "deepseek-chat"
```

| 配置项 | 是否必填 | 说明 |
|--------|---------|------|
| `database.url` | 是 | 数据库地址，默认使用本地 SQLite |
| `ai.api_key` | 否 | DeepSeek API 密钥，AI 分析功能需要 |
| `ai.base_url` | 否 | API 地址，可替换为 OpenAI/通义千问等兼容接口 |
| `ai.model` | 否 | 模型名称 |

## 常见问题

**Q: 截图识别不准确怎么办？**

A: 使用手机原图截图（不要裁剪/压缩），确保基金列表完整。识别结果可在确认导入前手动修改。

**Q: 基金代码没有自动匹配上？**

A: 系统通过基金名称模糊搜索匹配，部分简写名称可能匹配失败。确认导入时可手动输入正确的基金代码。

**Q: 总市值和手动求和差一分钱？**

A: 不会。系统使用高精度 Decimal 计算，四舍五入到分，前端与后端数值一致。

**Q: AI 分析报错？**

A: 请检查 `config.yaml` 中 `ai.api_key` 是否已正确配置，并确认网络可访问 API 地址。

**Q: 如何部署到服务器？**

A: 打包项目文件：
```bash
zip -r data-finance.zip . \
  -x "*.db" "__pycache__/*" ".venv/*" "venv/*" \
  -x ".DS_Store" "config.yaml" "uploads/*" "*.zip"
```
上传解压后安装依赖、编辑配置、启动服务即可。

**Q: 如何清空所有数据重新开始？**

A: 删除 `fund_manager.db` 文件，重启服务会自动创建新数据库。或在页面上使用「一键清空」按钮。
